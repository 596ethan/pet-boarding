# **1. 项目概述**
本项目是一个面向宠物寄养门店的小型管理系统。系统目标不是做复杂平台，而是完成一个清晰、完整、可演示的业务闭环：从主人建档、宠物建档、订单创建，到入住分配房间、记录照护，再到退房完成。

系统由三部分组成：鸿蒙前端、管理后台、Java 后端。鸿蒙前端负责现场操作和简洁展示；管理后台负责数据管理与查询；Java 后端负责接口、业务规则和数据库操作。

|**一句话定义**<br>这是一个以“增删改查 + 完整流程”为核心、以鸿蒙原生组件实现简洁 UI 的宠物寄养管理小型项目。|
| :- |

# **5. 完整业务流程**
系统必须至少能完整跑通以下流程：

1. 管理员或店员登录系统；
1. 新增宠物主人信息；
1. 新增宠物档案并关联主人；
1. 创建寄养订单；
1. 为订单分配房间并办理入住；
1. 寄养期间记录喂食、遛放、清洁、用药或异常情况；
1. 办理退房；
1. 订单状态变为已完成，后台可查看整条历史记录。

|[登录]<br>`   `↓<br>[新增主人]<br>`   `↓<br>[新增宠物]<br>`   `↓<br>[创建订单]<br>`   `↓<br>[入住登记 + 分配房间]<br>`   `↓<br>[照护记录]<br>`   `↓<br>[退房结算]<br>`   `↓<br>[订单完成]|

# **6. 鸿蒙前端需求**
## 6\.1 定位
鸿蒙前端是现场操作端，不追求大而全，重点是页面整洁、操作快捷、流程顺畅。前端应使用鸿蒙原生组件实现统一风格的 UI，突出小型项目的原生开发能力。
## 6\.2 页面清单

|**页面**|**主要功能**|**优先级**|
| :-: | :-: | :-: |
|登录页|账号密码登录|P0|
|首页|今日入住、今日退房、寄养中数量、快捷入口|P0|
|主人列表/表单页|主人增删改查|P0|
|宠物列表/表单页|宠物增删改查|P0|
|订单列表/表单页|订单增删改查|P0|
|入住页|选择订单、分配房间、确认入住|P0|
|照护记录页|新增与查看照护记录|P0|
|退房页|订单退房、状态完成|P0|

## 6\.3 UI 设计原则
- 整体风格简洁，颜色不宜过多；
- 列表页以搜索框 + 列表项 + 新增按钮为主；
- 表单页以分组表单为主，避免字段过多拥挤；
- 首页使用卡片式布局展示关键数据；
- 关键状态使用标签 Badge 展示；
- 危险操作使用弹窗确认；
- 操作按钮位置固定、文案明确。
## 6\.4 建议使用的鸿蒙组件
- List / Scroll：展示主人、宠物、订单、照护记录列表；
- TextInput / Search：搜索与表单输入；
- Button：新增、保存、删除、入住、退房等操作；
- Select / Picker：选择宠物类型、订单状态、房间、时间；
- Tabs / Navigation：页面切换；
- Dialog / Toast：确认提示与反馈；
- Badge / Text：状态展示。
# **7. 管理后台需求**
管理后台不是 Java 接口本身，而是管理员使用的数据管理界面。它的目标很明确：把主人、宠物、订单、房间、照护记录这些数据管理起来。

|**模块**|**功能点**|**说明**|
| :-: | :-: | :-: |
|登录|登录 / 退出|管理员入口|
|主人管理|新增、编辑、删除、查看、搜索|完整 CRUD|
|宠物管理|新增、编辑、删除、查看、搜索|完整 CRUD|
|订单管理|新增、编辑、删除、查看、状态变更|核心模块|
|房间管理|新增、编辑、删除、查看状态|支持分配与占用管理|
|照护记录管理|新增、编辑、删除、查看|支持按订单查看|
|基础统计|寄养中数量、今日入住、今日退房、空闲房间|简要展示即可|

# **8. Java 后端需求**
Java 后端负责提供接口、处理业务规则和完成数据库操作。对这个小项目来说，后端重点不是高并发，而是规则正确、结构清楚、接口稳定。

|**模块**|**职责**|
| :-: | :-: |
|认证模块|管理员登录、令牌校验|
|主人模块|主人信息 CRUD 接口|
|宠物模块|宠物信息 CRUD 接口|
|订单模块|订单 CRUD、入住、退房、状态流转|
|房间模块|房间 CRUD 与占用状态变更|
|照护记录模块|照护记录 CRUD 接口|

## 后端必须处理的关键规则：
- 入住时应校验房间是否已被占用；
- 退房后房间状态应恢复可用；
- 已完成订单不应继续新增普通照护记录；
- 删除主体数据时要考虑是否存在关联订单。
# **9. 数据库设计（MVP）**
小型项目不需要太多表。MVP 只保留最核心的数据表即可。

|**表名**|**说明**|**关键字段**|
| :-: | :-: | :-: |
|user|管理员用户|id, username, password|
|owner|宠物主人|id, name, phone, remark|
|pet|宠物档案|id, owner\_id, name, type, breed, age, weight, temperament|
|room|房间/笼位|id, room\_no, status|
|order|寄养订单|id, owner\_id, pet\_id, room\_id, status, checkin\_time, checkout\_time|
|care\_record|照护记录|id, order\_id, type, content, record\_time|
